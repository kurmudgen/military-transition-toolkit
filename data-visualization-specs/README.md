## Specification Format

Each specification follows a comprehensive template including:

1. **Purpose & Use Case** - Goals, audience, key messages
2. **Data Requirements** - Sources, sample data, processing needs
3. **Visualization Specifications** - Chart type, dimensions, colors, typography
4. **Layout Description** - Wireframes and component placement
5. **Color Palette** - Exact hex codes with accessibility compliance
6. **Typography** - Font families, sizes, weights
7. **Visual Elements** - Axes, data points, labels, legends, annotations
8. **Interactive Elements** - Hover states, click actions, animations
9. **Content Specifications** - Titles, insights, labels, attributions
10. **Design Guidelines** - Do's and don'ts
11. **Example Mockup Description** - Detailed visual description
12. **Usage Examples** - Blog, social media, email contexts
13. **Production Notes** - Creation method, time estimates, complexity
14. **Quality Checklist** - Pre-launch verification
15. **Variations** - Alternative versions for testing
16. **Related Visualizations** - Complementary specs
